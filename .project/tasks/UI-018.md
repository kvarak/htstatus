# UI-018: Standardize Modal Content Width Constraints

## Problem Statement

Player modal content uses inline max-width styles (450px) applied to individual sections instead of leveraging the established CSS logical component architecture. This creates scattered styling that's harder to maintain and bypasses the project's component-based design system.

## Current Issues
- Inline styles: `style="max-width: 450px"` scattered across multiple sections
- Bypasses established CSS logical architecture in app/static/css/
- Inconsistent with ui-components.css component patterns
- Maintenance overhead when width standards change
- Poor separation of concerns (presentation mixed with markup)

## Implementation

### Create CSS Classes
Add to `app/static/css/ui-components.css`:
```css
/* Modal content width constraints */
.modal-content-constrained {
  max-width: 450px;
}

.modal-section-constrained {
  max-width: 450px;
}
```

### Update Player Template
Replace inline styles in `app/templates/player.html`:
```html
<!-- From: -->
<div class="row mb-3" style="max-width: 450px;">
<table class="table-custom" style="max-width: 450px;">

<!-- To: -->
<div class="row mb-3 modal-section-constrained">
<table class="table-custom modal-section-constrained">
```

### Update Chart Container
Replace chart container inline styles:
```html
<!-- From: -->
<div class="player-chart" style="margin: 1rem 0; padding: 0.5rem; background: #f8f9fa; border-radius: 0.25rem; max-width: 470px; max-height: 320px;">

<!-- To: -->
<div class="player-chart modal-section-constrained" style="margin: 1rem 0; padding: 0.5rem; background: #f8f9fa; border-radius: 0.25rem; max-height: 320px;">
```

## Acceptance Criteria

✅ Create CSS classes following ui-components.css patterns
✅ Replace all inline max-width styles with CSS classes
✅ Maintain identical visual layout and responsiveness
✅ Follow established CSS logical component architecture
✅ No template rendering issues or visual regressions
